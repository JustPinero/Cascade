# Component & A11y Rules

- Server components by default. Add `"use client"` only for useState/useEffect, event handlers, or browser APIs — and never import Prisma, `fs`, or `child_process` in a client component.
- Tailwind for all styling — no inline `style=`, no CSS modules.
- Interactive elements must be real elements: `<button>`/`<a>`, not clickable `<div>`s. Keyboard operability comes free when you do.
- Every form control gets a label; every meaningful image gets `alt`; icon-only buttons get `aria-label`.
- Status/health indicators (fleet health, dispatch outcomes) must not rely on color alone — pair color with text or an icon.
- Use `loading.tsx` for route-level loading skeletons and `error.tsx` for per-segment error boundaries.
- Use `generateMetadata` in server components only.
