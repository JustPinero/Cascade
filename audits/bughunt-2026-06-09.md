## CRITICAL

### C1. Shell injection via project name in tmux placeholder
**`lib/claude-dispatcher.ts:453-456`** â€” `queuedPlaceholderCmd(projectName)` only strips single-quotes:
```ts
const safe = projectName.replace(/'/g, "");
return `echo '[queued: ${safe}] waiting for concurrency slot'; exec bash -i`;
```
Project name flows in from `j.project.name` (lines 696, 839) and reaches `execSync` via `createPaneGrid` â†’ `tmux new-session -d -s ${TMUX_SESSION} -n "projects-1" '${escapeForTmux(placeholder)}'` (line 474). A project named `foo$(rm -rf $HOME)` or `foo\nrm bar` executes after the single-quote strip leaves `;`, `$()`, backticks, and newlines intact. `lib/validators.ts:56` already exports `sanitizeForShell` (`/[\`$\\;"'|&<>(){}!\n\r]/g`) â€” should be applied here.
**Fix:** replace line 454 with `const safe = sanitizeForShell(projectName);`.

## HIGH

### H1. `deploy-monitor.ts` external fetches have no timeout
**`lib/deploy-monitor.ts:23` (Vercel) and `:82` (Railway)** â€” neither fetch passes a `signal`. If Vercel or Railway hang, the request stalls the server route that called `getVercelStatus`/`getRailwayStatus` until Node's default socket timeout (~minutes). This is reached by `/api/integrations/deploy-status` (referenced from `app/projects/[slug]/page.tsx:179`).
**Fix:** wrap each in `AbortController` with ~10s timeout, matching `lib/deploy-health.ts:27-35` pattern.

### H2. Client-side overseer chat fetch can hang the browser tab
**`app/components/overseer-chat.tsx:462`** â€” `fetch("/api/overseer/chat", ...)` has no `AbortController`. If the route's 60s server-side abort fails to close the SSE body, the reader loop at line 492 (`while (true) { reader.read() }`) can spin indefinitely. The server route IS guarded (`app/api/overseer/chat/route.ts:309-310`), but the client should have its own watchdog so a hung connection doesn't leave `streaming=true` forever.
**Fix:** add an `AbortController` with ~90s timeout, pass to fetch + clear on `done`.

## MEDIUM

### M1. Unguarded `JSON.parse(lesson.tags)` in JSX rendering
**`app/knowledge/page.tsx:197` and `app/knowledge/[category]/page.tsx:79`** call `JSON.parse(lesson.tags)` directly inside the `<LessonCard tags={...} />` prop, no try/catch. A single malformed `tags` row (manual seed, mid-migration, etc.) throws synchronously and crashes the entire knowledge page render. **Fix:** mirror the pattern from `lesson/[id]/page.tsx`.

### M2. `lib/anthropic-feature-check.ts:219` JSON.parse of LLM output unguarded
