# Phase 20 — Delamain voice

## Why
Justin's been chasing the Jarvis-from-Iron-Man pattern: a coding-
focused AI assistant that *speaks*. Until now Delamain is text-only
with optional sound effects (start/end pings) and voice INPUT
(SpeechRecognition for dictation). What's missing: voice OUTPUT —
Delamain speaking responses back.

## Stack choice
**Web Speech API (`speechSynthesis`)**, not cloud TTS. Reasons:
- Free, instant, works offline, no API key plumbing
- Already permissioned by the browser (no extra UX dialogs)
- Voice variety is "good enough" — every modern browser ships a
  handful of built-in voices, plus the user's OS voices on Mac/Win
- Future cloud TTS (ElevenLabs/OpenAI) can swap in behind the same
  `speak()` interface — out of scope for v1

## Customizable surface
Per Justin's ask:
- **Master on/off toggle** (also a quick toggle in the chat header
  for mid-session muting)
- **Voice selector** (dropdown of `speechSynthesis.getVoices()`)
- **Rate** slider (0.5–2.0×)
- **Pitch** slider (0.5–2.0×)

## Sub-requests
1. **20.1 — Voice settings + speak() helper**
   - Extend `OverseerSettings` with `voiceEnabled`, `voiceURI`,
     `voiceRate`, `voicePitch`. Defaults: enabled=false, URI=null
     (browser default), rate=1.0, pitch=1.0.
   - New `lib/speak.ts`: `speak(text, options)`, `cancel()`,
     `listVoices()`. Wraps the browser API; tests mock it.
2. **20.2 — Voice settings UI**
   - New "Voice" panel in `app/settings/page.tsx`. Toggle, voice
     dropdown (filtered by language), rate + pitch sliders, "Test
     voice" button that speaks a sample sentence.
3. **20.3 — Chat integration**
   - On response complete (after streaming finishes) → speak the
     final assistant text if `voiceEnabled`.
   - On new message send → `cancel()` any in-flight speech.
   - Quick mute toggle in the chat header (icon button, persists
     to settings).

## Acceptance
- Settings round-trip persists voice prefs in `localStorage`.
- `speak()` is a no-op when `voiceEnabled === false` or running
  server-side (no `speechSynthesis`).
- Cancel works mid-utterance (test with mocked API).
- Validate.sh green.

## Out of scope
- Cloud TTS provider integration
- Streaming TTS (speak as text streams, not after complete)
- Voice cloning / custom voice IDs
- Per-message TTS (whole response or nothing)
