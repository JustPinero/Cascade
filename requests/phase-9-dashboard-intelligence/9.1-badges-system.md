# Request 9.1 — Project Badges System

## Objective
Add a flexible badge system to projects. Badges are visual labels
on tiles that communicate status beyond health and progress:
deployed, awaiting-review, testing, client, versioned.

## Acceptance Criteria → Test Mapping

| Criterion | Test |
|-----------|------|
| Badges stored as JSON array on Project | schema has badges String field |
| PATCH endpoint accepts badges array | PATCH /api/projects/[slug] updates badges |
| Valid badges enforced | only allowed badge values accepted |
| Badges render on project tile | tile shows badge pills |
| Client badge renders with cyan accent | client badge has distinct styling |
| Empty badges array shows nothing | no badge UI when array empty |
| Badges visible but not cluttering tile | badges are small pills below progress bar |

## RED Phase
- getBadges returns empty array when no badges set
- PATCH accepts valid badge values
- PATCH rejects invalid badge values
- Tile component renders badge pills from project data

## GREEN Phase
- Add `badges String @default("[]")` to Project schema
- Update PATCH allowlist with badge validation
- Update project-tile.tsx to render badge pills
- Badge styles: deployed=success, client=cyan, testing=amber,
  awaiting-review=accent, versioned=space-400
