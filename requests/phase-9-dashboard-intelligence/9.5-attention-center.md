# Request 9.5 — Attention Center

## Objective
A single count in the sidebar that tells the developer: "X things
need you right now." Consolidates human tasks, blocked projects,
CI failures, and deploy-down into one number.

## Acceptance Criteria → Test Mapping

| Criterion | Test |
|-----------|------|
| Sidebar shows attention count badge | sidebar renders count next to nav |
| Count includes: pending human tasks | attention count includes task count |
| Count includes: blocked projects (health=blocked) | attention count includes blocked count |
| Count includes: CI failures | attention count includes CI fail count |
| Count includes: deploys that are down | attention count includes deploy-down count |
| Zero attention shows no badge | badge hidden when count is 0 |
| Badge pulses when count > 0 | pulse animation on non-zero count |
| Clicking badge navigates to /tasks | badge links to tasks page |
| Count refreshes on window focus | uses same focus-refresh as dashboard |

## RED Phase
- getAttentionCount returns 0 when nothing needs attention
- getAttentionCount sums tasks + blocked + CI fails + deploy-down
- Sidebar renders badge with count
- Badge hidden when 0

## GREEN Phase
- Create GET /api/attention endpoint (returns consolidated count)
- Create app/components/attention-badge.tsx
- Wire into sidebar.tsx (client component wrapper needed)
- Pulse animation from existing CSS (pulse-warning)
- Refresh on visibilitychange like dashboard
