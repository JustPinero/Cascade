# Request 9.3 — CI Status Integration

## Objective
Show GitHub Actions CI status on project tiles. If CI is failing,
the developer sees it on the dashboard without checking email.

## Acceptance Criteria → Test Mapping

| Criterion | Test |
|-----------|------|
| Fetch latest CI run status from GitHub API | getCIStatus returns pass/fail/none |
| Projects without githubRepo return "none" | getCIStatus handles missing repo |
| CI failure shows indicator on tile | tile renders CI fail badge |
| CI passing shows no extra indicator | tile is clean when CI passes |
| CI status cached for 5 minutes | repeated calls within TTL return cache |
| CI check runs during scan | scan pipeline includes CI status |

## RED Phase
- getCIStatus returns "pass" for successful run
- getCIStatus returns "fail" for failed run
- getCIStatus returns "none" for no repo
- getCIStatus caches results for TTL

## GREEN Phase
- Create lib/ci-status.ts using `gh run list` or GitHub API
- Cache results in memory with 5-minute TTL
- Wire into scan pipeline
- Update project-tile.tsx to show CI fail indicator
- Store CI status in healthDetails JSON

## Files to Touch
- lib/ci-status.ts (new)
- lib/ci-status.test.ts (new)
- lib/project-import.ts
- app/components/project-tile.tsx

## Dependencies
- None

## Notes
- Use `gh run list --repo owner/repo --limit 1 --json conclusion`
  for simplicity. Falls back gracefully if gh CLI not installed.
- Only check repos that have githubRepo field set.
