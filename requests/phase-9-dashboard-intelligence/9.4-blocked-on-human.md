# Request 9.4 — Blocked-on-Human Indicator

## Objective
When a project has pending human tasks, show it on the tile.
"This project is waiting on YOU, not on Claude." Different from
[NEEDS ATTENTION] which means Claude is stuck.

## Acceptance Criteria → Test Mapping

| Criterion | Test |
|-----------|------|
| Tile shows pending human task count for project | tile renders task count badge |
| Count only includes pending tasks (not done) | query filters by status=pending |
| Zero pending tasks shows nothing | no indicator when no tasks |
| Indicator is visually distinct from health | uses different color/position than health dot |
| API returns human task count per project | GET /api/projects includes humanTaskCount |

## RED Phase
- Projects API returns humanTaskCount field
- humanTaskCount only counts pending tasks
- Tile renders count when > 0
- Tile hides indicator when 0

## GREEN Phase
- Update GET /api/projects to include pending human task count
- Update ProjectTileData interface
- Update project-tile.tsx with "X tasks for you" indicator
- Position below badges, subtle but visible

## Files to Touch
- app/api/projects/route.ts (add humanTaskCount to response)
- app/components/project-tile.tsx
- app/page.tsx (pass through to tile)

## Dependencies
- None (human tasks system already exists)
