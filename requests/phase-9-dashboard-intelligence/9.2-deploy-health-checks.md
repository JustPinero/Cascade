# Request 9.2 — Deploy Health Checks

## Objective
Ping deployed project URLs during scans to verify they're up.
If a deployed app is down, tile goes red and notification fires.

## Acceptance Criteria → Test Mapping

| Criterion | Test |
|-----------|------|
| Health check pings URL from deploymentInfo | checkDeployHealth fetches URL and returns status |
| 200 response = healthy | checkDeployHealth returns healthy for 200 |
| Non-200 or timeout = down | checkDeployHealth returns down for 500/timeout |
| Only checks projects with deployed badge + URL | scan skips projects without deploymentInfo.url |
| Down status sets health to warning | project health reflects deploy status |
| Notification fires when deploy goes down | activity event created for deploy-down |
| Health check has 10s timeout | fetch aborts after 10 seconds |

## RED Phase
- checkDeployHealth returns healthy for 200 response
- checkDeployHealth returns down for non-200
- checkDeployHealth returns down on timeout
- checkDeployHealth skips when no URL configured

## GREEN Phase
- Create lib/deploy-health.ts with checkDeployHealth()
- Wire into scan pipeline (importSingleProject and importProjects)
- Create deploy-down activity event when status changes
- Fire desktop notification via activity feed polling
- Add deploy status to healthDetails JSON
