# Request 9.6 — Optional Deadlines with Countdown

## Objective
Projects can have an optional deadline. When set, the tile shows
a countdown. Del factors deadlines into sprint planning and
morning briefings. Deadlines auto-resolve when progress hits 100%.

## Acceptance Criteria → Test Mapping

| Criterion | Test |
|-----------|------|
| Deadline stored as nullable DateTime on Project | schema has deadline field |
| PATCH endpoint accepts deadline date | PATCH sets deadline |
| Project detail page has date picker for deadline | detail page renders date input |
| Tile shows countdown when deadline set | tile renders "3d left" or "due today" |
| Tile goes amber within 3 days of deadline | countdown styling changes at 3 days |
| Tile goes red when overdue | overdue styling is danger color |
| Deadline auto-resolves when progress=100% | completed project shows "completed on time" or "completed X days late" |
| Deadline extension logs activity event | changing deadline creates event |
| Del reads deadline in overseer prompt | buildOverseerSystemPrompt includes deadline |
| Del mentions deadlines in morning briefing | briefing prompt includes deadlines |
| Del can set deadline via chat | [DEADLINE] tag parsed from overseer response |

## RED Phase
- Deadline field nullable, returns null when not set
- formatCountdown returns "3d left" for 3 days out
- formatCountdown returns "due today" for today
- formatCountdown returns "2d overdue" for past deadlines
- Deadline auto-resolves message when progress=100

## GREEN Phase
- Add `deadline DateTime?` to Project schema
- Update PATCH allowlist
- Add date picker to project detail page
- Add countdown display to project-tile.tsx
- Color: default when >3d, amber when ≤3d, red when overdue
- Add deadline to overseer system prompt per-project
- Add [DEADLINE] tag parsing in overseer-chat.tsx
  Format: [DEADLINE] project-slug — YYYY-MM-DD
- Auto-resolve: when progressScore=100, show completion status
- Log activity event on deadline change
