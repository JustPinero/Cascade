# Request 13.5 ‚Äî Session-state endpoint

## What was wrong
`propose_dispatch` (Phase 12C.2) writes to `workingMemory.proposedDispatches`,
but no consumer reads it. The dashboard parses `[DISPATCH]` text tags
from chat output. Senior code review flagged this as half-shipped.

## What ships
- `GET /api/overseer/session-state` ‚Äî returns `{sessionId, startedAt, closedAt, activeFlow, workingMemory}` for a given date (today by default).
- Tests covering: empty session, populated state, date scoping, default-today, propose_dispatch use case.

## Why this matters
The dashboard can now fetch structured outputs from Cascade in a single
request, instead of (a) parsing `[DISPATCH]` tags from chat text, or
(b) reading the database directly. Whenever the dashboard is updated to
consume `workingMemory.proposedDispatches`, the `[DISPATCH]` tag bridge
in `TOOL_PATH_SYSTEM_PROMPT` can be deleted.
