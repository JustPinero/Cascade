# Request 13.3 — UX + cosmetics

## What ships
- `sseFromText` accepts `{model?, messageId?}` and defaults to honest
  values (`claude-sonnet-4-6`, dynamic timestamp id) instead of the
  `cascade-feature-check` leftover from Phase 11.
- Truncation surface (`formatTruncationSurface` in route.ts): when
  the loop bails, the response now lists the tools that were called
  with counts ("query_project (5×), update_session_memory (4×) ...")
  plus a hint to narrow the question. Replaces the generic "I hit my
  iteration limit" line.
- `buildDefaultRegistry()` cached at module load — was being rebuilt
  on every request. Pure module-level singleton (registry is
  request-independent).
- `closeSession` is left exported but unused; documented intent in
  the helper's docstring (future cron job concern).

## Acceptance
- Lint + tsc clean
- All 680 tests still pass

## Out of scope
- Removing `body.useTools` flag entirely (kept as no-op for
  backward compat; the test that previously asserted "false bypasses"
  was reframed in 12F as "doesn't break")
- Streaming the terminal text token-by-token (one SSE chunk for now)
