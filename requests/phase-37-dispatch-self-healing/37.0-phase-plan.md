# Phase 37 ‚Äî Self-healing dispatch lifecycle [36.A1] [36.A2]

Closes the HIGH finding from `audits/design-review-2026-06-11.md`: queue slots
are keyed by `project.path` and only released by the Stop-hook webhook, so a
missing hook, a path-spelling mismatch, or two dispatches of the same project
can wedge the fleet (concurrency 1 on <16GB boxes). Restarts strand `queued`
Dispatch rows forever.

## What ships

1. **Queue keyed by `idempotencyKey`** (`lib/dispatch-lifecycle.ts`). Each
   dispatch holds its own slot ‚Äî same-project dispatches no longer collide,
   and releases can't miss on path byte-differences.
2. **Watchdog releases by `idempotencyKey`** (`lib/dispatch-watchdog.ts`) ‚Äî
   drops the per-row `project.path` lookup entirely.
3. **Webhook releases by `idempotencyKey`** when the Dispatch row is found;
   the legacy (key-less) path falls back to releasing the newest
   queued/started row's key for the project, plus the old `projectPath`
   release for transition safety.
4. **Boot reconciliation** ‚Äî `reconcileOrphanedDispatches(prisma)`: any row
   still `queued` at process start is orphaned by definition (its enqueue
   closure died with the previous process); flip to `failed` with
   `errorMessage: "orphaned by server restart"`. `started` rows are left
   alone ‚Äî their session may still be running and the webhook/watchdog owns
   them. Wired into `startDispatchWatchdog()` (before the first tick) and
   `scripts/run-dispatch-watchdog.ts`.
5. **Quick wins riding along:** `countTestFiles` skips `dist`/`build`/
   `.next`/`coverage`/`out` ([36.A3] mitigation); env-prefix + single-quote
   shell helpers deduped in the dispatcher ([36.C1]).
