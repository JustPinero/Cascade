# Phase 15 — Follow-ups from the third review

The third senior code review surfaced 9 findings, several of which
were issues with prior fixes that overpromised. This phase addresses
all of them in one bundle.

## What ships

| # | Finding | Fix |
|---|---|---|
| 1 | 14.4's $transaction wrapping a single update was a no-op | Comment honestly; revert to plain update; document last-writer-wins as acceptable for a cache |
| 2 | TZ fix half-deployed (server accepts sessionDate; dashboard never sent it) | Dashboard now sends `localToday()` (`Intl.DateTimeFormat("en-CA")`) in every chat POST |
| 3 | `isValidSessionDate` regex too lax — `2026-13-99` passed | Also assert `new Date(s + "T00:00:00.000Z")` is a real Date |
| 4 | `closeStaleSessions` had no production caller | New `scripts/run-stale-session-cleanup.ts`, invoked from `scripts/start.sh` next to the version watcher; closes sessions older than 30 days |
| 5 | "Tag contract test" wasn't really a contract — example string was hardcoded in the test, not extracted from the SP | New exported const `DISPATCH_TAG_EXAMPLE` in route.ts; SP literal interpolates it; test imports it from the route. Drift on either side now fails. |
| 6 | No route-level test of any write tool (only unit tests) | New end-to-end test: model emits `update_session_memory` tool_use → route runs the helpers → `chatSession.update` is called with the merged workingMemory containing the patched value |
| 7 | Dead `useTools: true` in 4 test bodies | Stripped via a small Python pass |
| 8 | Summarizer fallback had no telemetry — stderr only | `ActivityEvent({eventType: "compressor-fallback"})` recorded on each fallback, with droppedCount + error message |
| 9 | `get_session_state` inconsistent with write tools on closed sessions | Documented as intentionally read-through (archeology is legitimate); test added asserting the policy |

705 → 709 (+4). Some assertions tightened in place rather than added.
