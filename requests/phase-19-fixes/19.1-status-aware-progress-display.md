# Request 19.1 — Status-aware progress display

## Why
Delamain reported (in chat, 2026-04-29): completed and deployed
projects show their raw `progressScore` prominently, which is
phase-completion math that never hit 100%. PingThings (46%),
HR Hero (14%), labwebsite (42%), CON-CORE (92%). A shipped product
displaying 14% looks broken.

## Acceptance Criteria → Test Mapping

| Criterion | Test |
|---|---|
| `lib/project-display.ts` exports `getCompletionDisplay(project)` returning a discriminated union | unit test |
| `status === "deployed"` returns `{kind: "shipped", label: "Deployed", score}` | unit test |
| `status === "complete"` returns `{kind: "shipped", label: "Complete", score}` | unit test |
| Any other status returns `{kind: "in-progress", score}` | unit test, all four other statuses |
| Helper accepts only the fields it needs (status, progressScore) — keeps consumers loosely coupled | type test (input shape) |
| `app/components/project-tile.tsx` renders a "Shipped" badge instead of the progress bar when `kind === "shipped"`; hovering shows the raw score in a `title` attribute | component code path verifiable by snapshot/manual |
