# Phase 19 — Bugs surfaced by Delamain

Two bugs Delamain flagged in chat that need to actually land.

## 19.1 — Status-aware progress display
PingThings (46%), HR Hero (14%), labwebsite (42%), CON-CORE (92%)
are all marked `complete` or `deployed` — but the dashboard surfaces
the raw `progressScore` (phase-completion math) which never reached
100%. A shipped product showing 14% looks broken.

Fix: a pure `lib/project-display.ts` helper that returns either
`{kind: "shipped", label}` for completed/deployed projects or
`{kind: "in-progress", score}` otherwise. Project tile + roadmap
render the result. Raw score still available via title tooltip on
shipped projects (transparency without misleading prominence).

## 19.2 — Engineer-channel writeback
Today, when Del emits `[ENGINEER]` in chat output, nothing parses or
persists it. Justin asked Del "post this to the engineer channel"
and Del's note never made it — buried in `ChatMessage` rows.

Fix: pure `lib/engineer-tag-parser.ts` that extracts `[ENGINEER]`
blocks from terminal text. Route handler fires-and-forgets a
`/api/engineer-channel` POST per message after the tool loop returns.
Mirrors the `[DISPATCH]` tag-parser pattern.
