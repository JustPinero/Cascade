# Phase 22 — Lead-stall guardrails + placeholder portrait

## Why
Two unrelated needs converging in one phase:

1. **Delamain reported a real bug last sprint** (engineer brief
   2026-04-29): a lead Claude Code session in `--teammate-mode tmux`
   tried to spawn 3 teammates via parallel `Agent` calls. The bound
   iTerm session was dead. One Agent call returned the explicit
   error; the other two returned `[Tool result missing due to
   internal error]`. The lead ended its turn silently. The
   conversation idled 19 hours before the user noticed.

   The deepest fixes (Agent error surfacing, lead recovery
   behavior) live inside Claude Code itself — filed separately as
   a bug report. This phase ships the **Cascade-side guardrails**:
   pre-flight validation, idle detection, sprint-prompt hardening.

2. **Theme Pack work needs a default placeholder portrait** so
   every theme degrades gracefully if its asset is missing or
   404s. Ships first because it unblocks future theme work without
   waiting on Leonardo.

## Sub-requests

### 22.1 — Default placeholder portrait
- New `app/components/portrait.tsx` — `<Portrait src alt className/>`
  wrapper around `<img>`. If `src` is empty/null/undefined OR the
  image errors at load, falls back to an SSR-safe inline SVG
  (`<DefaultPortrait/>`) — neutral, theme-agnostic.
- Replace the four `<img>` portrait tags in
  `app/components/overseer-chat.tsx` and the previews in
  `app/settings/page.tsx` with `<Portrait/>`.
- Tests: rendering with valid src, with empty src, with src that
  errors.

### 22.2 — iTerm session validator
- `lib/iterm-session-validator.ts` exports
  `isITermSessionAlive(sessionId): Promise<boolean>`. Uses
  `osascript` to query iTerm; resolves false on any error.
- Pure function (subprocess injectable for tests).
- Tests: alive returns true, dead returns false, malformed input
  rejects, subprocess failure returns false (defensive).

### 22.3 — `dispatchTeam` validation gate
- Before launching the lead session, validate the iTerm session
  is reachable. Refuse to dispatch if not — surface a clear error
  to the API caller.
- Tests at the unit level (mocked validator).

### 22.4 — Idle / partial-team detector
- `lib/team-config-scanner.ts` — pure helper that reads
  `~/.claude/teams/*/config.json` and returns broken-team
  diagnostics: members with `tmuxPaneId === ""`, teams with no
  recent task activity.
- Wired into a scheduled scan (similar to closeStaleSessions).
  Fires `ActivityEvent({eventType: "team-stalled"})` for hits.
- Tests with fixture team config files.

### 22.5 — Sprint-prompt hardening
- Append explicit "if any tool batch errors, emit user-visible
  text before yielding" rules to the lead's system prompt in
  `lib/claude-dispatcher.ts:dispatchTeam`'s sprintPrompt.
- Workaround for the silent-yield Claude Code bug we filed.
- No new tests (the prompt is a string; the behavior change is
  observed by real lead sessions).

### 22.6 — Validate + handoff
- `scripts/validate.sh` green.
- Update `.claude/handoff.md` with Phase 22 summary + which
  guardrails caught what during testing.
- Note the lead-stall bug report status in handoff for future
  reference.

- Rolling back Claude Code's team config writes on spawn failure

- The Theme Pack registry itself — Phase 23, after Leonardo
