# Phase 33 — Route-boundary tests

Closes audit finding **[30.D2]**. Only 2 of 41 routes (`overseer/session-state`, `preflight`) had route-level tests at the start of the day. The dispatcher and webhook lib layers are well-covered, but the HTTP boundary on the highest-blast-radius mutating endpoints is uncovered.

## Scope (this phase)

| Target | Coverage focus |
|--------|---------------|
| `lib/dispatch-lifecycle.ts` | Direct unit tests — Phase 23.2 core path with no direct tests. queued→started→completed/failed transitions, idempotencyKey collision behavior, spawn rejection, watchdog deadline. |
| `app/api/projects/launch/route.ts` | Happy path (valid payload), 400 validation cases, 500 error surface. |
| `app/api/projects/[slug]/dispatch/route.ts` | Single-project dispatch route — slug lookup, dispatch invocation, ready/not-ready paths. |
| `app/api/dispatch/team/route.ts` | Agent team launch — input validation, Windows-error path (platform mock). |
| `app/api/webhook/session-complete/route.ts` | Smoke: 400 on missing body, 200 happy path with idempotencyKey, escalation parse + activity event side-effect. |

## Out of scope (intentional, defer to a future phase)

| Target | Why deferred |
|--------|--------------|
| `app/api/overseer/chat/route.ts` | Streaming SSE + tool-use loop + slash commands. Test surface is enormous; a smoke-only test would be misleadingly small. Worth a dedicated request. |
| `app/components/overseer-chat.tsx` (1093 LOC) | Heavy DOM + speech recognition + TTS + session memory wiring. Worth a dedicated UI-testing slice. |
| Remaining 36 routes | Lower blast-radius; the top-5 cover the highest-risk surface. |

## Approach

Each test file is independent. Delegate to 5 parallel general-purpose agents, each with the route-test pattern from `app/api/overseer/session-state/route.test.ts` as a reference, and explicit instructions to:

- Only WRITE the test file. Do not modify route source.
- Use the dispatch rig (`tests/harness/dispatch-rig.ts`) for routes that exercise the dispatcher.
- Run the new test to verify it passes before reporting back.

| # | Criterion | Test |

| 3 | `app/api/projects/[slug]/dispatch/route.test.ts` exists | route test |
