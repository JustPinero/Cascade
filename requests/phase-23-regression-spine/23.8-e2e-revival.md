# Request 23.8 — E2E Revival + CI Integration

## Objective

The 5 existing Playwright tests in `e2e/` are dead weight: `.github/workflows/ci.yml` gates them with `if: false`. Either fix or replace. This slice does the cheaper thing — delete the stale tests and ship 3 fast browser smokes that exercise the actual critical paths a user touches: dashboard load, dispatch trigger, Overseer chat round-trip. Plus wire the Phase 23 jobs (`scenario`, `snapshot`, `evals`) into CI so this whole phase actually gates merges.

After this slice, there are no half-disabled CI jobs and no stale tests living in the repo.

## GREEN Phase (implementation)

Files to create:

- `e2e/setup/seed.ts` — pure script. Connects to the test DB (driven by `process.env.DATABASE_URL`), pushes schema if needed, inserts fixtures: 1 healthy project (`medipal`), 1 project with `health: blocked` and a recent `[NEEDS ATTENTION]` activity event, 1 ChatSession + 1 ChatMessage from yesterday for the Overseer chat smoke. Idempotent — re-runs do not double-insert.
- `e2e/setup/seed.test.ts` — test 6.
- `e2e/setup/mock-anthropic.ts` — small fetch interceptor (or a Playwright `route.fulfill`) that intercepts `api.anthropic.com` requests during the chat smoke and returns a canned SSE-shaped response. Used by `overseer-chat-roundtrip.spec.ts`.
- `e2e/dashboard-loads.spec.ts` — Playwright test. Boots the dev server (or uses the running one), navigates to `/`, waits for project tiles, asserts at least the seeded medipal tile is present.
- `e2e/dispatch-button.spec.ts` — navigates to `/projects/medipal`, clicks the dispatch button, asserts the API call fires, the row appears in `Dispatch` table (via API or DB poke), and the UI shows a confirmation.
- `e2e/overseer-chat-roundtrip.spec.ts` — opens Overseer chat, sends "fleet status", waits for the assistant message to render. Uses the mock interceptor for Anthropic.
- `e2e/__test__/config.test.ts` — test 5.
- `e2e/__test__/perf.test.ts` — test 11.
- `.github/__test__/ci.test.ts` — tests 7-10. Reads ci.yml as YAML.

Files to modify:

- `playwright.config.ts` — set `webServer` to start `pnpm dev` against the test DB. Set `use.baseURL` to `http://localhost:3000`. Use a fresh `DATABASE_URL` like `file:./test-e2e.db` driven by env. Run seed before tests via `globalSetup`.
- `.github/workflows/ci.yml`:
  - Remove `if: false` from the e2e job
  - Add a `scenario` job after `test`: `pnpm test tests/scenarios/`
  - Either add a `snapshot` job running `pnpm test lib/__tests__/prompt-snapshots.test.ts`, or document that the existing `test` job covers it (snapshots are vitest, run with regular `pnpm test`)
  - Confirm the `evals` workflow from 23.6 is present alongside (separate file, separate triggers — already shipped in 23.6)
- `package.json`:
  - Add `"test:e2e:seed": "tsx e2e/setup/seed.ts"` if useful
  - Confirm `"test:e2e": "playwright test"` is the entry point

Files to delete:

- The 5 stale Playwright specs in `e2e/` (names captured at audit time — likely `app.spec.ts`, `health.spec.ts`, etc.). Audit and delete in the same commit as the new specs land.

## Out of scope

- **Visual regression testing.** No screenshot diffs in this slice. Future option if UI churn becomes a problem.
- **Mobile / multi-browser e2e.** Chromium-only, single viewport.
- **End-to-end testing of dispatch lifecycle including real Claude Code CLI.** Out — that's hermetic-impossible. The dispatch-button smoke just confirms the API call fires; the dispatch lifecycle is covered by scenario tests (23.5).
- **Replacing Playwright with Puppeteer / Cypress.** The choice is locked.

## Definition of done

- 0 stale Playwright tests in the repo.
- 3 new Playwright tests pass locally and in CI.
- CI runs e2e, scenario, snapshot tests, and evals (separate workflow) on every PR.
- Total e2e wall-clock under 90s.
