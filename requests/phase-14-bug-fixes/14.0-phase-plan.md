# Phase 14 — Bug fixes from the second code review

Phase 13 closed every actionable item from the first review. The
second review surfaced more — most around test gaps, a handful of
real bugs that production traffic would hit.

## Sub-requests

- **14.1** TZ-aware session binding — `body.sessionDate?` overrides
  server UTC. Closes the "9pm Eastern crosses into a new session at
  midnight UTC" cliff.
- **14.2** Summarizer failure fallback — Haiku going down should not
  500 the whole chat. Compressor catches and falls back to raw
  truncation.
- **14.3** `propose_dispatch` slug validation — verify the project
  exists before recording the proposal. Tool error otherwise.
- **14.4** Compressor concurrency — wrap read+update in
  `$transaction` so parallel requests don't overwrite each other.
- **14.5** Dead-code cleanup — drop the no-op `body.useTools` branch
  and stale comments.
- **14.6** Tag/regex contract test — assert that the SP-documented
  `[DISPATCH]` format actually parses through the dashboard regex.
- **14.7** Route error-path tests — missing API key, rate limit, exact
  tool count.
- **14.8** `closeStaleSessions` helper — makes the `closedAt`
  invariant enforceable; future cron can call it.

## Intentionally deferred

- **`commit_dispatches` effect tool.** The half-shipped finding from
  the review (#9). Adding the tool today would let the model fire
  dispatches autonomously without the user clicking Execute Sprint —
  that's worse, not better. Either the dashboard needs to call
  `commit_dispatches` on Execute Sprint click, or the model needs
  explicit autonomy boundaries. Either is its own scope.

## Exit criteria
- All 🟥 and 🟧 review items either fixed, tested, or explicitly
  deferred with rationale.
- `validate.sh` green.
- Test count grows; existing tests stay green.
