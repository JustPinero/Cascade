# Phase 29 — Multi-pane wt layout for batch dispatch

## Why

Phase 26 ships batch dispatch on Windows as N independent tabs in the user's current wt window. Functional but not dense — Justin asked for "as close to the tmux grid feel as possible" during the Phase 26 conversation. Split-panes within one dedicated wt window get us there.

## Design

wt supports targeting a **named window**: `wt -w <name> new-tab` creates the window if it doesn't exist, otherwise targets the existing one. Combined with `split-pane`, this gives a single-window-with-N-panes layout that survives the dispatcher's async-spawnFn architecture without needing inter-spawn coordination.

- `maybeCreatePaneGrid` returns synthetic targets shaped `"<batch-name>:<index>"` on Windows (e.g. `["cascade-1730000000:0", ":1", ":2"]`).
- `launchForJob` on Windows parses the target. Index 0 → `wt -w <name> new-tab` (creates the batch window). Index > 0 → `wt -w <name> split-pane` (adds a pane to that window).
- `wt`'s default split direction has a "stairs" pattern (always splits the active pane). v1 accepts this — Justin can rearrange. A future polish slice can chain `move-focus` or use `tiled` arrangement.

Single dispatch (`dispatchClaude`) is unchanged — still goes through `launchInTerminal` with `-w 0` (current/any window).

## Acceptance criteria → tests

| # | Criterion | Test |
|---|-----------|------|
| 1 | On Windows, `dispatchAll` spawns the first job with `wt.exe ... new-tab` | `lib/claude-dispatcher.windows.test.ts` extension |
| 2 | On Windows, `dispatchAll` spawns subsequent jobs with `wt.exe ... split-pane` | same |
| 3 | All spawns target the same `-w <batchName>` window | same |
| 4 | `dispatchClaude` (single) still uses `-w 0 new-tab` (unchanged from Phase 26) | regression |
| 5 | macOS / Linux flows unaffected | existing tests still green |

## Out of scope

- Layout polish (grid vs stairs). Default wt behavior ships.
- Window naming surfaced to user. The batch name is internal; the user sees a wt window with their project panes.
- Inter-pane focus / navigation customization.
