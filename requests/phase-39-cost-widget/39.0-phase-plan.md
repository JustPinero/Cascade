# Phase 39 тАФ Dashboard cost widget [P8]

Design-review recommendation P8: `AnthropicUsageEvent` already records
per-call token + cache counters, but spend is only visible by mentally
multiplying numbers on `/observability/cache`. A solo dev running an
autonomous fleet on a metered API wants "what did today cost" at a glance.

## What ships

1. **`lib/observability/usage-summary.ts`** тАФ `getUsageSummary(prisma, {since})`:
   aggregates events since a timestamp into `{calls, inputTokens,
   outputTokens, cacheReadTokens, cacheCreationTokens, hitRate,
   estimatedCostUsd, hasUnknownModels}`. Pricing table (USD/MTok,
   June 2026): Sonnet 4.6 $3/$15, Haiku 4.5 $1/$5, Opus 4.8 $5/$25;
   cache reads 0.1├Ч input, 5m cache writes 1.25├Ч, 1h writes 2├Ч. When the
   5m/1h split is absent but `cacheCreationInputTokens` > 0, it's priced
   as 5m. Unknown models price at Sonnet rates and set `hasUnknownModels`
   so the UI can mark the figure as approximate.
2. **`GET /api/usage/summary`** тАФ today's summary (local midnight).
3. **`<CostWidget />`** тАФ compact header badge: `$0.43 today ┬╖ 12 calls ┬╖
   cache 72%`. Mount fetch + 60s poll; renders nothing until first
   successful fetch.

## Out of scope

  observability page).
- Historical charting; budgets/alerts.
- Live pricing lookup тАФ constants are dated and commented; update when
  models change (greppable via MODEL_PRICING).
