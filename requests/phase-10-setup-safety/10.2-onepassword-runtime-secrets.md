# Request 10.2 — 1Password as Runtime Secret Source

## Objective
Cascade's own runtime secrets (ANTHROPIC_API_KEY, any future ones) live
in 1Password. `.env` contains only `op://` references — no plaintext
secrets on disk. Dev and prod start scripts wrap with `op run` so
the Next.js process receives resolved values at fork time; spawned
subagents inherit from its environment. Removes the backup-to-1P
`populate-vault.sh` flow in favor of 1P-as-source-of-truth.

  - `resolveOpRef(ref: string): Promise<string>` — shells `op read`, returns value or throws descriptive error
  - Startup guard `assertOpReady()` — throws with install/signin instructions if prerequisites missing
- Rewrite `.env.example` with `op://` references for `ANTHROPIC_API_KEY`

## Notes
- Required prereq: `op` CLI installed AND signed in. No plaintext fallback.
- The env vars resolved by `op run` are inherited by all child processes spawned by Next.js, including the `claude` subagents dispatched via `claude-dispatcher.ts`. No changes needed in dispatcher code.
- 1P Desktop app is recommended for biometric re-auth but not required — CLI-only signin flow works.
- Post-implementation, `op signin` must be valid at Cascade startup. If the session expires mid-run, Cascade keeps working (env already resolved into process memory); only a restart would re-require a valid session.
