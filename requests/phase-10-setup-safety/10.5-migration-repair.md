# Request 10.5 â€” Migration Repair (stale paths + orphans)

## Objective
Make any existing Cascade install bring itself to the present on a new
machine. Today, after a Macâ†’Windows migration, every Project row's `path`
points at the old machine; the dashboard is fully broken even though the
fix is mechanical. This request adds a versioned, idempotent repair flow.

The fix has two parts:
1. **A + B (already covered):** projects on disk get repaired by running
   the existing `POST /api/projects/scan` â€” `lib/project-import.ts` upserts
   by slug and writes the new path. Surface this as a one-click repair.
2. **C (new):** projects in DB with no on-disk copy anywhere. Offer per-row
   actions: clone from GitHub, archive, delete, skip.

## Bucket reference (for tests)
- **A.** On disk + in DB (slug match) â€” repaired by existing scan.
- **B.** On disk + not in DB â€” created by existing scan.
