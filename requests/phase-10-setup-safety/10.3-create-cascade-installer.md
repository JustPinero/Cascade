# Request 10.3 — create-cascade Installer

## Objective
Deliver `npx create-cascade` — a one-command bootstrap that takes a new
user from empty machine to running Cascade with: prereq check, repo
clone, deps, 1P vault bootstrap, hook wiring, DB init, smoke test.
Ships as a sibling repo `github.com/JustPinero/create-cascade` published
to npm as `create-cascade`. Pattern: `create-t3-app`, `create-expo-app`.

## Acceptance Criteria → Test Mapping

| Criterion | Test |
|-----------|------|
| CLI detects OS; refuses pure Windows with WSL2 guidance | mock `platform()` → expect exit 1 + guidance message |
| Prereq check identifies missing `node` (≥22), `pnpm`, `claude`, `tmux`, `op` | unit test per tool with mocked `which` |
| Missing prereq list is printed with per-OS install one-liners | snapshot test on output |
| On WSL2 host, offers to write `~/.wslconfig` with safe defaults | prompt flow test; file written contains `autoMemoryReclaim=gradual` |
| User confirms install path (default `~/Code/cascade`) | prompt default assertion |
| Repo cloned to chosen path via `gh repo clone` or `git clone` | exec spy with clone URL |
| `pnpm install` runs in the cloned dir | exec spy |
| Prompts `op signin` if session is expired | checks `op vault list`, prompts on failure |
| Prompts user for Anthropic API key (input hidden) | prompt flow test |
| Creates "Cascade" vault + "Cascade Runtime" item with the key | spy on `op item create` |
| Writes `.env` with `op://Cascade/Cascade Runtime/anthropic_api_key` reference | file content assertion |
| Runs `scripts/install-hooks.ts` to wire the Stop hook | spy on exec |
| Runs `pnpm db:push` then `pnpm db:seed` | exec spy in order |
| Smoke test: starts Cascade in background, polls `/api/health`, asserts 200 | integration test with real server |
| Smoke test: triggers a $0.00002 Haiku ping to Anthropic via `ANTHROPIC_API_KEY` to verify live auth | integration test with real API (gated by `--with-live-check` flag by default ON) |
| Prints final URL, offers to open in browser | exec spy for `open` / `xdg-open` / `explorer.exe` |
| Every failure mode prints a single-screen error with remediation | snapshot tests per failure path |
| Exit codes: 0 success, 1 prereq missing, 2 1P failure, 3 clone failure, 4 install failure, 5 smoke failure | unit tests per path |

## RED Phase
Scaffold `create-cascade` repo with vitest test suite. Write failing
tests for: `detectOS`, `checkPrereqs`, `writeWslConfig`, `promptInstallPath`,
`cloneRepo`, `runPnpmInstall`, `ensureOpSignin`, `promptAndStoreApiKey`,
`writeEnvFile`, `installClaudeHooks`, `initDb`, `smokeTest`, `printSuccess`,
plus an e2e test that runs the full flow against a tmp dir with mocked
externals. Run `vitest` → fail.

## GREEN Phase
- Scaffold new repo: `create-cascade` — TypeScript, vitest, bin entry
- Dependencies: `@clack/prompts` (interactive prompts), `execa` (shell), `zod` (validation), `chalk` (colors minimal)
- Steps module, each pure-ish function with a clear signature
- Orchestrator wires steps in order, emits progress
- Failure surfaces exit codes consistently; stderr carries remediation
- Publish workflow: npm publish pipeline via GitHub Actions, semver
- Local dev: `pnpm link --global` for testing

## Notes
- `@clack/prompts` is the current idiomatic prompt lib (used by Vite, Astro, T3)
- Smoke-test API call: use Haiku 4.5 (cheap), single "hi" prompt, assert 2xx response. $0.00002 per install attempt.
- Installer does NOT install prereqs automatically. It detects and prints one-liners. Respects the user's system, avoids sudo.
- Pure Windows is rejected early with clear "install WSL2 first" message pointing at Microsoft docs.
- Biometric vs master-password signin is a UX concern only — installer works with either.
- Focus test failures from 10.4 feed back into installer polish — expect a small iteration.
