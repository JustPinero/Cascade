# Request 41.3 — Publish-Safety & Secret-Hygiene Audit

## Objective
Detect the incident class that cost a git history rewrite on
2026-07-07: ephemeral session files tracked in public repos, and
credentials embedded where they don't belong. Runs as part of fleet
health; findings become escalations/HumanTasks automatically.

## What it must catch (all observed in the wild this week)
1. Ephemeral session files TRACKED in git: `.claude/handoff.md`,
   `.claude/kilroy-channel.md`, `.claude/sessions/`,
   `.claude/settings.local.json`, non-example `.env*`.
2. Secret-pattern matches in TRACKED files (reuse/port the pattern set
   from the coqui-kickoff secret-scan: sk-ant-, sk-, ghp_, github_pat_,
   AKIA, private-key blocks, xox*, plus postgres://user:pass@ URLs and
   sbp_/sntrys_ tokens).
3. Credentials embedded in `.claude/settings.local.json` permission
   strings (7 of 21 fleet projects had this — tokens, service-role
   JWTs, DB passwords inside allow-rule text).
4. Public-repo awareness: severity escalates when the repo's origin is
   public (derive via `gh repo view --json visibility`, cached; treat
   unknown/no-remote as private).

## Acceptance Criteria → Test Mapping

| Criterion | Test |
|-----------|------|
| Tracked ephemeral file detected | fixture repo with tracked handoff.md → finding with file + class |
| Tracked secret pattern detected | fixture with a fake sk-ant key in a tracked file → finding |
| settings.local.json embedded credential detected | fixture with postgres://u:p@ in an allow string → finding (severity: credential) |
| Public repo escalates severity | same finding, visibility=public → severity high; private → normal |
| Clean project yields zero findings | clean fixture → [] |
| Findings create HumanTasks idempotently | run twice → one HumanTask per distinct finding, category `credential` or `review` |
| Secrets never stored/logged verbatim | finding text contains a redacted form (first 10 chars + …), assert raw value absent from DB row and logs |
| Surfaced in fleet health | health payload includes publishSafety summary per project |

## RED Phase

(rules: tests use throwaway dirs/DBs, never dev.db).

## GREEN Phase
