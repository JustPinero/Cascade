# Request 41.7 — Infrastructure-Version Health Dimension

## Objective
Per-project infrastructure state as a first-class health signal: which
coqui-kickoff plugin version applies, whether v3.5 machinery remnants
remain, and whether the workspace trust dialog is accepted (untrusted
workspaces silently ignore project allow-lists in dispatched sessions
— observed on sharpesanimalhouse during the migration).

## Signals to compute per project
1. **Plugin version**: read `~/.claude/skills/coqui-kickoff/.claude-plugin/plugin.json`
   version (machine-level, same for all projects — surface once, not
   per-project, but include in payload).
2. **Migration state**: v3.5 remnants = project-local
   `.claude/skills|agents|commands` containing any plugin-provided
   names (the kit's Step 1 list — these SHADOW the plugin), or
   session-context/secret-scan hooks still in project settings.json.
   States: `v4`, `v3.5-remnants` (list them), `no-kickoff` (never had
   machinery).
3. **Workspace trust**: detect from `~/.claude.json` project entries
   (verify the exact key at Prime — e.g. hasTrustDialogAccepted); if
   not reliably determinable, report `unknown`, never guess.

## Acceptance Criteria → Test Mapping

| Criterion | Test |
|-----------|------|
| Plugin version read and reported | scratch skills dir fixture → version string in payload |
| Missing plugin reported as not-installed | no plugin dir → `not-installed`, no crash |
| v3.5 remnant detection by exact names | fixture project with a shadowing skill (e.g. .claude/skills/bughunt) → `v3.5-remnants` listing it |
| Custom-named machinery is NOT a remnant | fixture with only custom names (sharpes pattern) → `v4` |
| no-kickoff projects classified | fixture with no .claude machinery → `no-kickoff` |
| Trust state read or honestly unknown | fixture ~/.claude.json with and without the entry → accepted / unknown |
| Dashboard/health payload carries the dimension | health payload includes infraVersion block per project |
| Briefing flags shadowing remnants | briefing payload lists projects with `v3.5-remnants` |
