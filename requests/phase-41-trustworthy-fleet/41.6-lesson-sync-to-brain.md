# Request 41.6 — Lesson Sync to kilroy-brain

## Objective
What Kilroy learns on one machine should exist on the others. The
knowledge harvester currently writes lessons into Cascade's local DB
(machine-local by definition). Mirror harvested lessons into the
private brain repo at `~/kilroy-brain/playbook/lessons/` so the
existing brain pull/install flow carries them across machines.

## Design
- On harvest of a `[LESSON]`, additionally write/update a markdown
  file in the brain lessons dir: one file per lesson, kebab-case slug,
  frontmatter (source project, date, tags), body = lesson content.
- Brain path configurable via env (`KILROY_BRAIN_PATH`, default
  `~/kilroy-brain`); if the dir is absent (machine without the brain),
  skip silently with a log line — never fail the harvest.
- Do NOT git-commit from Cascade — the brain's own harvest.sh/manual
  flow owns commits. Cascade only writes files.
- Dedup: re-harvesting the same lesson updates the same file (slug
  match), no duplicates.

## Acceptance Criteria → Test Mapping

| Criterion | Test |
|-----------|------|
| Lesson harvest writes a brain markdown file | fixture lesson → file exists at scratch brain path with frontmatter + body |
| Slug is stable and filesystem-safe | lesson with punctuation/emoji in title → valid kebab-case filename |
| Same lesson re-harvested updates, not duplicates | harvest twice → one file, updated mtime/content |
| Missing brain dir skips gracefully | env pointing at nonexistent dir → harvest succeeds, no file, one log line |
| DB behavior unchanged | existing harvester tests still pass |
| No git operations performed | assert no `git` shell-out from the sync path (spy on the shell boundary) |
