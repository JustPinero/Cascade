# Request 12B.1 — Fleet-visibility read tools

## Objective
Replace the four largest blocks of the current Overseer system prompt
(active project list, recent activity, per-project session logs,
dispatch-outcome stats) with on-demand read tools. Each tool uses the
12A.2 framework. The tool path runs alongside the legacy SP-injection
path; default flip happens in 12B.3.

## Tools

### `query_projects(filter?)`
- Input: `{ status?: string[]; health?: string[]; includeBackburner?: boolean; limit?: number }`
- Output: array of project summaries (slug, name, status, health, phase, progressScore, businessStage, lastActivityAtISO)
- Default: excludes backburner + archived; limit 25; ordered by `lastActivityAt desc`

### `get_recent_activity(filter?)`
- Input: `{ projectSlug?: string; eventType?: string; limit?: number }`
- Output: `[{eventType, summary, projectSlug?, projectName?, createdAtISO}]`
- Default: limit 10; latest first

### `get_session_logs(input)`
- Input: `{ slug: string; limit?: number }`
- Output: `{found: bool, slug: string, logs: [{summary: string, capturedAtISO?: string}]}`
- Default limit 1; reads via `lib/session-reader`

### `get_dispatch_outcomes(filter?)`
- Input: `{ projectSlug?: string; mode?: string; limit?: number }`
- Output: `{ totals: {[mode]: {total, success, blocker, successRate}}, recentFailures: [...] }`
- Default limit 50
