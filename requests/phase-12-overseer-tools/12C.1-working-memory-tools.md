# Request 12C.1 — Working memory write tools + route session binding

## Objective
Give the Overseer structured working memory: the ability to record
confirmed answers, set/clear the active flow, and query the session
state. This is the foundation that fixes the inventory-walk bug —
confirmed values stop living only in raw conversation prose.

Bind every tool-path request to a `ChatSession` so `ctx.sessionId` is
always populated.

## Tools
### `update_session_memory`
- ctx requires `sessionId`
- Input: `{ patch: Record<string, unknown> }`
- Output: `{ newState: Record<string, unknown> }`
- Deep-merges patch into `chatSession.workingMemory`. Throws via tool
  error if session is closed or sessionId missing.

### `set_active_flow`
- ctx requires `sessionId`
- Input: `{ flow: "inventory_walk" | "dispatch_planning" | "incident_triage" | null }`
- Output: `{ flow: string | null }`
- Whitelisted values; null clears.

### `get_session_state`
- ctx requires `sessionId`
- Input: `{}`
- Output: `{ sessionId, activeFlow, workingMemory }`
- Read-only; surfaces the canonical session state for self-reference.

## Route changes
- Before running the tool loop, call `getOrCreateSession(prisma, today)` to bind a session
- Attach `ctx.sessionId` so working-memory tools work
- No persistence of messages yet (Phase 12D adds that alongside flow tracking)
