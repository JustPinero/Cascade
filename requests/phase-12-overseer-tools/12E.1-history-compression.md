# Request 12E.1 — History compression safety net

## Objective
For long conversations, replace older raw turns with a cached summary
so the message log doesn't grow unbounded. The compressor is a
SAFETY NET — `workingMemory` (12C.1) remains the canonical store for
confirmed facts. Compression just keeps the message array
manageable on multi-hour sessions.

## What ships
- `ChatSession.compressedHistory: String?` (JSON: {summarizedThroughMessageCount, summary})
- `lib/chat-history-compressor.ts`
  - `compressMessagesForSession(prisma, sessionId, messages, opts)` — pure helper, returns either the input unchanged (≤ threshold) or `[summary message, ...recent N messages]`
  - Caches the summary on the session; reuses it on subsequent calls when the older portion hasn't grown
  - Re-summarizes when the older portion grows beyond the cached count
  - Tolerates malformed JSON in `compressedHistory` (recomputes)
  - `defaultSummarizer(apiKey)` — Haiku-backed implementation; pure injection so tests use a mock
- Route integration: tool-path request runs the compressor before runToolUseLoop. Default threshold = 25, keepRecent = 10.
