# Request 12B.3 — System-prompt slim + flip useTools default + slash precedence

## Objective
Make the tool-use path the default for `/api/overseer/chat`. The legacy
SP-injection flow remains reachable for one transition cycle via
explicit `body.useTools: false` (removed entirely in Phase 12F).
Slash commands still take precedence over both paths.

## Changes
- Expanded `TOOL_PATH_SYSTEM_PROMPT` to advertise the full read-tool
  registry (8 tools after 12B.1 + 12B.2), tag output formats
  (`[DISPATCH]`, `[REMINDER]`, `[HUMAN TODO]`, `[PLAYBOOK]`,
  `[ENGINEER]`), and dispatcher-style guidance. Length still
  constant per turn — no per-request data baked in.
- Reordered the route handler: slash commands fire first, then the
  tool path (default), then the legacy path (`useTools: false`).
- Updated `route.tools.test.ts`: dropped the "does NOT run when
  unset" assertion (semantics flipped), added "DOES run when
  unset", expanded the SP test to assert the full read-tool surface
  is advertised to the model.

## Acceptance
| Criterion | Test |
|---|---|
| Tool path runs when `useTools` is unset | route.tools.test.ts |
| Tool path runs when `useTools: true` is explicit | route.tools.test.ts (existing) |
| Tool path is bypassed when `useTools: false` is explicit | route.tools.test.ts |
| Slash commands fire before the tool path regardless of useTools | route.feature-check.test.ts + route.feature-propose.test.ts |
| The tool path's SP advertises all 8 read tools | route.tools.test.ts |

## Out of scope
- Streaming the terminal text in the tool path (still one SSE chunk)
- Removing the legacy `buildOverseerSystemPrompt` + streaming branch
  (Phase 12F)
- Migrating `[DISPATCH]` / `[REMINDER]` / `[HUMAN TODO]` to
  structured tools (Phase 12C)
