# Request 12F.1 — Decommission legacy SP-injection paths

## Objective
Close out Phase 12 by removing the legacy paths that the tool migration
made obsolete.

## What was removed
- `buildOverseerSystemPrompt()` — the 280-line per-turn SP builder
  that pulled project list / activity / session logs / yesterday
  summary / outcomes / playbook / engineer channel into a single
  ~6K-token snapshot. All of that data is now fetched on demand
  via tools (Phase 12B).
- `formatTimeAgo()` — only used by the function above.
- The `useTools !== false` branch + the streaming `fetch` fallback.
  Tool path is now the only path.
- Unused imports: `fs/promises`, `path`, `getSessionLogs`.
- The `[REMINDER]`, `[HUMAN TODO]`, `[PLAYBOOK]`, `[ENGINEER]` tag
  formats from the SP. The structured tools (`create_reminder`,
  `create_human_todo`, etc.) cover them and the dashboard does not
  parse them from chat output.

## What was kept
- The `[DISPATCH]` tag format in the SP. The dashboard
  (`app/components/overseer-chat.tsx:298`) parses it from chat
  output to render Execute Sprint buttons. The SP now instructs
  the model to ALSO call `propose_dispatch` so the canonical record
  is structured; the tag is a UI bridge until the dashboard reads
  `workingMemory.proposedDispatches` directly (separate scope).
