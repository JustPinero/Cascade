## Objective

Instrument the Overseer's tool-use loop so every `registry.execute()` call writes a `ToolCallEvent` row capturing what was called, with what input, how long it took, and whether it succeeded. Expose the data via a new tool `get_tool_call_stats` for in-conversation queries and via a flat `/observability/tools` page for visual scanning.

After this slice, "the model never calls `get_engineer_messages` even in inventory walks" stops being a hunch â€” it's a query. Tool-design questions become measurable.

Schema and design are in `references/tool-call-observability.md`.

- **Truncation of huge tool outputs in the chat transcript.** This slice only truncates the `input` field stored on the telemetry row; the actual chat transcript (`ChatMessage`) is unchanged.
