# Request 24.1 — Outcome-Conditioned Dispatch (propose-only)

## Objective

Wire the Overseer to read its own dispatch history before proposing a new dispatch. The Overseer surfaces *"based on the last 5 outcomes I'd recommend continue instead of audit"* — the developer still triggers. Auto-apply is explicitly out of scope; that's a future opt-in toggle once propose-only proves itself.

This slice ships:
1. A new tool `query_outcome_history({ slug, windowDays? })` returning per-mode counts, success rates, recurring signals, and a one-line heuristic-generated `summary`.
2. A short addition to `TOOL_PATH_SYSTEM_PROMPT` instructing the Overseer to call this tool before `propose_dispatch`.
3. Three eval fixtures that catch regression on the new behavior.

Design and tool surface are specified in `references/outcome-conditioned-dispatch.md`.

## GREEN Phase (implementation)

Files to create:

- `lib/overseer-history-summary.ts` — pure function `generateSummary(byMode, recentTimeline)` returning a one-line string. All heuristics live here, testable in isolation.
- `lib/overseer-history-summary.test.ts` — tests 6-9.
- `lib/overseer-tools-outcome-history.ts` — exports the tool definition. Reads from `Dispatch` joined to `DispatchOutcome` via `dispatchId` (post-23.2). Falls back to `DispatchOutcome.projectSlug` for orphaned outcomes pre-23.2.
- `lib/overseer-tools-outcome-history.test.ts` — tests 1-5.
- `evals/scenarios/overseer/repeated-audit-no-signals.json` — preconditions seed 5 prior audit outcomes with empty signals. Asserter: tool sequence includes `query_outcome_history` then `propose_dispatch({mode: "continue"})`. Final text matches `/audit/i`.
- `evals/scenarios/overseer/continue-keeps-failing.json` — preconditions: 3 prior continue outcomes with `outcome: blocker`. Asserter: proposes `investigate`.
- `evals/scenarios/overseer/first-dispatch-no-history.json` — no prior outcomes. Asserter: `query_outcome_history` is called once, `propose_dispatch` with whatever mode the rules suggest, final text does not contain "history" or "previous".
- Recordings for the three new fixtures in `evals/recordings/overseer/<scenario>/<hash>.json`.

Files to modify:

- `lib/overseer-tools-registry-default.ts` — register the new tool.
- `app/api/overseer/chat/route.ts` — extend `TOOL_PATH_SYSTEM_PROMPT` with:
  ```
  # Outcome-conditioned proposals
  Before calling propose_dispatch, ALWAYS call query_outcome_history({ slug }) for the
  project you're about to dispatch. If the history suggests the developer's preferred mode
  isn't producing useful signals (e.g. 3 consecutive audits with no findings), surface that
  and propose the alternative mode in your text. The developer still triggers; you advise.
  ```
- `lib/__tests__/prompt-snapshots.test.ts` — update snapshot to reflect the new tool def + system prompt addition.

## Dependencies

- 23.4 (Caching rollout) — adding a tool changes `toAnthropicTools()` output; the cache_control marker shifts to the new last tool. One-time cache miss; snapshot test catches this so the change is reviewable.

## Out of scope

- **Auto-apply.** Propose-only must prove itself first. Auto-apply lives behind a future playbook flag (`outcomeConditionedAutoApply: true`) and routes only to mode `continue` when history is unambiguous.
- **Surfacing outcome history in the dashboard UI.** The Overseer is the surface; a `/projects/<slug>/history` page is a different feature.
- **Per-mode dispatch templates** (e.g. "audit + outcome history → audit-with-context-from-history"). The Overseer adapts its proposal in text; the dispatched session prompt is unchanged.
- **Reading outcomes from sibling projects** (cross-project pattern detection). Within-project history only for now.
- **Removing or deprecating `propose_dispatch`'s default rules**. The model still calls `propose_dispatch`; the new tool informs which mode it picks.

## Definition of done

- The new tool ships and is exposed in the default registry.
- The system prompt change is live and snapshotted.
- All 14 tests pass (unit + integration + 3 evals).
- A real Overseer turn against a project with audit-heavy history mentions the audit pattern in its proposal text. (Manual smoke; note in handoff.md.)
