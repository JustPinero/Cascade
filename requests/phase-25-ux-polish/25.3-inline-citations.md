# Request 25.3 — Inline Knowledge Citations

## Objective

The Overseer's harvest loop is the most distinctive thing Cascade has. Today its value is invisible: when the Overseer answers using a `KnowledgeLesson`, the lesson disappears into paraphrase. Anthropic's Citations API attaches structured citations to response text — `cited_text` does not count toward output tokens, citations are guaranteed to point at real spans, and citation accuracy is materially better than prompt-engineered citation tags.

This slice ships:

1. A new tool `query_knowledge_with_citations({ query, topN? })` that runs the matcher and returns matched lessons formatted as a citations-enabled custom-content document block.
2. A small route handler intercept that injects the returned document block into the next Anthropic request alongside the standard tool result.
3. A response-side citation resolver that maps `start_block_index` back to lesson IDs.
4. A dashboard UI change to render inline citations as clickable `[L-42]` links to lesson detail pages.

This is Path B from `references/anthropic-citations.md` — the model decides when to fetch knowledge, keeping the cache prefix stable when knowledge isn't relevant.

## GREEN Phase (implementation)

Files to create:

- `lib/overseer-tools-knowledge-citations.ts` — exports the tool definition. The handler:
  1. Calls `matchIssueToLessons(query, lessons)` from `lib/knowledge-matcher.ts` (pre-existing) — returns ranked lessons.
  2. Builds the document block with `cache_control: { type: "ephemeral" }`.
  3. Returns the envelope `{ lessons, document }`.
- `lib/overseer-tools-knowledge-citations.test.ts` — tests 2-5.
- `app/api/overseer/chat/route.citations.test.ts` — tests 6-8.
- `app/components/overseer-chat.citations.test.tsx` — test 9.
- `evals/scenarios/overseer/knowledge-cited-response.json` + recording — test 10.

Files to modify:

- `lib/overseer-tools-registry-default.ts` — register `query_knowledge_with_citations`.
- `app/api/overseer/chat/route.ts`:
  - When the tool-use loop calls `query_knowledge_with_citations`:
    - Capture the document block from the envelope.
    - Inject it into the next user-role message: `[{ type: "tool_result", tool_use_id, content: stringifyEnvelope }, document]`. The `content` of `tool_result` summarizes the lessons textually so the model can use them even if it ignores citations.
    - Persist the block-index → lesson-id mapping in a request-scoped Map.
  - When parsing the response: scan each text block for a `citations` array; for each citation, look up `lessonId` via the Map; emit an SSE `event: citation` per citation with `{ lessonId, start_block_index, cited_text, ... }`.
- `app/components/overseer-chat.tsx`:
  - Subscribe to the `citation` SSE event; render inline `<a className="text-xs align-super text-blue-600 hover:underline" href={"/knowledge/" + lessonId}>[L-{lessonId}]</a>` after the corresponding cited text span.
  - The placement of the link follows the order of citation events; render them inline immediately after the text block they belong to.
- `evals/scenarios/overseer/knowledge-query.json` — update assertion to assert `query_knowledge_with_citations` is called (replaces the older permissive matcher assertion).
- `lib/__tests__/prompt-snapshots.test.ts` — update snapshot for the new tool def.

## Dependencies

- 25.2 (streaming) — citations are emitted via SSE events alongside text deltas; the streaming infrastructure must be in place.
- 23.4 (caching) — the document block carries `cache_control`, which combines with the existing system+tools cache.
- 23.6 + 23.7 (eval runner + fixtures) — for criteria 10-11.
- `lib/knowledge-matcher.ts` — pre-existing; this slice does not change matcher behavior, just consumes its output.

## Out of scope

- **Files API for the lesson corpus** (referenced in `references/anthropic-files-api.md`) — research only. Inlining lessons as custom-content documents is simpler at current corpus size. Adopt later if corpus grows past ~50K tokens regularly attached.
- **Citation feedback / "this citation was wrong"** — future feature.
- **Backwards-compatible behavior when the model paraphrases without calling the citations tool** — the model can still answer without citations; the inline-link rendering only fires when the structured citation arrives.
- **Citations on per-project chat or wizard chat** — Phase 25 ships on the Overseer chat path only; other endpoints are separate.
- **Image citations.** Anthropic Citations API does not support image citations as of writing; not applicable.
- **Replacing the `query_knowledge` tool entirely.** If a `query_knowledge` (non-citations) tool exists, leave it in place for backward compatibility within the conversation history. New chats will use the citations variant via system prompt guidance — but the prompt update is minimal.

- The Overseer harvest loop's value is now visible — Phase 25's distinctive UX outcome.
