# Request 25.2 — Streaming Overseer Responses

## Objective

The Overseer chat route currently buffers the full Anthropic response, then replays it as SSE for the dashboard. Real streaming gives:

- Time-to-first-token improvement (cache hits arrive in tens of ms, not seconds)
- Progressive tool-call visibility ("Del is calling `query_project`..." renders before the call resolves)
- Cleaner cancellation (aborts take effect mid-generation)
- Honest cache stats — `usage` arrives in `message_delta`, which the route can log immediately

This slice ships a streaming caller variant, plumbs it through the tool-use loop via an `onEvent` callback, and rewires the route to forward deltas to the client SSE channel. The loop body itself does not change — streaming is a transport concern handled inside the caller.

It also closes the open follow-up from 23.3 — usage logging for the streaming call sites (`/api/wizard/chat`, `/api/projects/[slug]/chat`) — by reusing the same `message_delta` pattern.

Event grammar and accumulator pattern are in `references/anthropic-streaming-tool-use.md`.

## GREEN Phase (implementation)

Files to create:

- `lib/overseer-tools-streaming.ts` (or co-locate in `lib/overseer-tools.ts`) — exports `defaultStreamingAnthropicCaller(apiKey)`. Uses fetch with `stream: true`, reads `Response.body` as a `ReadableStream`, parses SSE events, applies them to an accumulator (per the reference doc), forwards each event to `onEvent`. On `message_stop`, returns the assembled `AnthropicMessageResponse`. Logs `AnthropicUsageEvent` from `message_delta` before returning.
- `lib/streaming-accumulator.ts` — pure function that takes the current accumulator state plus an SSE event and returns the new state. Tests for the accumulator can run without any fetch or stream machinery.
- `lib/streaming-accumulator.test.ts` — accumulator unit tests (subset of test 11, 12).
- `lib/overseer-tools.streaming.test.ts` — tests 1-4, 11-13.
- `app/api/overseer/chat/route.streaming.test.ts` — tests 6-8.
- `app/api/wizard/chat/route.test.ts` — extend or create for test 9. (May already exist.)
- `app/api/projects/[slug]/chat/route.test.ts` — extend or create for test 10.

Files to modify:

- `lib/overseer-tools.ts`:
  - Widen `AnthropicCaller`'s options to include optional `onEvent: (e: StreamEvent) => void`.
  - Widen `ToolUseLoopParams` to include optional `onEvent`. Forward to `caller(params, { signal, onEvent })`.
  - The loop itself acts on the assembled `AnthropicMessageResponse` exactly as before. No behavior change.
- `app/api/overseer/chat/route.ts`:
  - Swap `defaultAnthropicCaller` for `defaultStreamingAnthropicCaller`.
  - Build a `TransformStream` that converts `onEvent` callbacks into client-facing SSE writes:
    - `text_delta` → forward as-is on the active text block
    - `tool_use` `content_block_start` → emit synthetic `event: tool_call_start\ndata: { name, id }` for the UI
    - Strip raw tool_use deltas from the client stream
    - `message_delta` → emit synthetic `event: usage` for client cache stats display (optional)
  - Replace the existing `sseFromText(final, ...)` call with the live transform — for the *tool-use* path. Keep `sseFromText` for the slash-command synthetic responses (no live model call → no streaming).
- `app/api/wizard/chat/route.ts`:
  - Already streams via `stream: true`. Wrap the response stream in a TransformStream that watches for `message_delta` and calls `logUsage` with `callSite: "wizard"`. Pipe through.
- `app/api/projects/[slug]/chat/route.ts`:
  - Same pattern as wizard. `callSite: "project.chat"`.

## Out of scope

- **Streaming on the briefing endpoint.** Briefing is one-shot and short; latency is fine. Skip.
- **Streaming on the feature proposer.** Burst calls; not user-facing latency-sensitive. Skip.
- **A general `streaming-fetch` helper used outside Anthropic calls.** This streaming work is Anthropic-specific. Generalize later if a use case appears.
- **Persistent connection / WebSocket.** SSE is correct for this; no upgrade.
- **Replacing the `sseFromText` synthetic stream.** Keep for slash-command paths where there's no live model.
