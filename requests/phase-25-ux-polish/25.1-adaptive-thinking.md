# Request 25.1 — Adaptive Thinking on Dispatch Decisions

## Objective

Sonnet 4.6 supports adaptive thinking natively — the model decides when reasoning helps and how much budget to spend, with no client-side `thinking` parameter required. The work in this slice is **not** to enable thinking (it's already on); it's to make sure Cascade's tool-use loop preserves thinking blocks correctly across tool turns. If we drop or mutate a thinking block when passing tool results back, the next turn errors. Today the loop preserves the full content array opaquely, which means thinking blocks ride through unchanged — but there's no test confirming that, and `ContentBlock` doesn't even include the type. This slice widens the type, adds tests proving round-trip preservation, and ships one eval that checks the model's dispatch proposals improve in quality (or at least stay coherent) when the dispatch turn has access to outcome history.

Manual extended thinking with `thinking: { type: "enabned", budget_tokens }` is **deprecated** on Sonnet 4.6 — see `references/anthropic-extended-thinking.md`. We do not enable it. If adaptive doesn't deliver, escalating to manual on Sonnet 4.5 is a future option.

## Out of scope

- **Manual extended thinking on Sonnet 4.5.** If adaptive doesn't deliver measurable proposal quality improvement, switching the dispatch-decision turn to Sonnet 4.5 with `thinking: { type: "enabled", budget_tokens: 5000 }` is a future option — and a separate slice. Adds a model split, breaks message caching across the boundary.
- **Surfacing thinking in the dashboard UI.** Adaptive thinking can be omitted from rendering (`display: "omitted"` not relevant since we're not setting `thinking` params). The chat UI sees the assistant's text turn; thinking blocks ride along internally and are invisible to the user. No UI change.
- **Logging thinking content for analytics.** Thinking blocks contain potentially-sensitive reasoning text. Don't log them.
- **Thinking on non-dispatch turns.** Adaptive on Sonnet 4.6 means the model decides per-turn; we don't gate it.
