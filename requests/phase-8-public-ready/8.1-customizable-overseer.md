# Request 8.1 — Customizable Overseer Identity

## Objective
Allow users to name their AI, upload a portrait, and personalize the
Overseer's identity. Default name is "Overseer" with the current
Delamain portrait. Everything currently hardcoded as "Delamain" becomes
configurable through a settings interface.

## Acceptance Criteria → Test Mapping

| Criterion | Test |
|-----------|------|
| Default Overseer name is "Overseer" | getOverseerSettings returns "Overseer" when no preference set |
| User can set custom name via settings | setOverseerSettings persists name to localStorage |
| Custom name renders in sidebar, chat header, portrait nameplate | OverseerChat renders custom name from settings |
| User can upload a single portrait image | upload endpoint saves to /public/overseer/ and updates settings |
| User can upload idle + talking pair | settings store both image paths |
| Default portrait is the current delamain.jpg | getOverseerSettings returns default portrait path |
| Single-image mode uses CSS glow for talking state | portrait uses glow animation when only one image provided |
| Two-image mode swaps images during streaming | portrait swaps src when streaming=true and talking image exists |
| Settings page shows Overseer Customization section | settings page renders name input, image upload, personality textarea |

## RED Phase
- getOverseerSettings returns defaults when localStorage empty
- setOverseerSettings persists and retrieves custom name
- getOverseerSettings returns custom portrait path after set
- getOverseerSettings distinguishes single vs dual image mode
