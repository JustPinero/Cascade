# Request 8.4 — Windows/WSL2 Support

## Objective
Make Cascade installable and functional on Windows via WSL2. Update
the dispatcher to detect the OS and use the appropriate terminal
launch method. Document the setup process.

## Acceptance Criteria → Test Mapping

| Criterion | Test |
|-----------|------|
| Dispatcher detects macOS vs Linux/WSL | detectPlatform returns "macos" or "linux" |
| macOS dispatch uses osascript (existing) | dispatchClaude uses osascript on darwin |
| Linux/WSL dispatch uses direct terminal | dispatchClaude uses shell exec on linux |
| tmux works identically on both platforms | tmux commands don't use macOS-specific flags |
| README has Windows/WSL2 setup section | README contains WSL2 installation steps |
| README has macOS setup section | README contains macOS-specific notes |
| Stop hooks use portable commands | hook commands work on bash (not zsh-specific) |
| Path handling uses path.join everywhere | no string concatenation for file paths |

## RED Phase
- detectPlatform returns correct value for current OS
- dispatchClaude selects correct launch method by platform

## GREEN Phase
- Create lib/platform.ts with detectPlatform()
- Update lib/claude-dispatcher.ts dispatch functions to branch on platform
- Linux dispatch: launch claude directly in current terminal or via tmux
  (no osascript, no Terminal.app)
- Audit all hook commands for bash portability
- Audit path handling across scanner, health engine, progress engine
- Write WSL2 setup guide in README
- Write macOS setup section in README

## Files to Touch
- lib/platform.ts (new)
- lib/platform.test.ts (new)
- lib/claude-dispatcher.ts
- README.md
- scripts/install-hooks.ts (verify portable)

## Dependencies
- None

## Notes
- WSL2 users access Cascade at localhost:3000 from Windows browser
- tmux works natively in WSL2 Ubuntu — no changes needed
- The osascript → Terminal.app flow is macOS-only
- On Linux/WSL2, dispatch launches claude directly in a tmux session
  (no osascript wrapper needed — tmux IS the terminal manager)
