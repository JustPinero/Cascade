# Phase 34 ‚Äî OverseerChat component smoke

Closes the residual UI piece of audit finding **[30.D2]**. `app/components/overseer-chat.tsx` (1093 LOC) had zero tests.

## What ships

Five smoke tests at `app/components/overseer-chat.test.tsx`:

1. Mounts without crashing; fires the initial `/api/overseer/session-state` + `/api/overseer/history` fetches.
2. Renders messages rehydrated from `/api/overseer/history`.
3. Survives a failing history fetch ‚Äî component still mounts, input still works.
4. Input field round-trips text typed by the user.
5. Renders the configured overseer name (smoke that the settings hook fires).

## Mocking surface

- `@/lib/sounds`, `@/lib/notify`, `@/lib/speak`, `@/lib/silence-detector`, `@/lib/dispatch-tag-parser`, `@/lib/local-today`, `@/lib/session-memory`, `@/lib/overseer-settings`, `@/app/components/portrait` all stubbed.
- `fetch` stubbed via `vi.stubGlobal` per test so each can return its own shapes for the history + session-state + chat endpoints.
- `window.SpeechRecognition` and `window.webkitSpeechRecognition` set to `undefined` so the component's speech path is a no-op.
- `Element.prototype.scrollTo` stubbed because jsdom doesn't implement it and the component's auto-scroll effect would throw.

## Out of scope

- Streaming SSE behavior (parsing `data:` lines into rendered text). Would need a custom ReadableStream mock; deeper than a smoke needs.
- Speech recognition / TTS interaction. Justin's voice flow is rich enough to warrant its own slice.
- Dispatch tag action UI (pending-actions panel, auto-execute path). Tested at the lib level via `dispatch-tag-parser` tests.
- Conversation Mode + silence detector wiring. Heavy state machine; its own slice.
